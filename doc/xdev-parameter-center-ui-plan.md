# XDEV 参数中心页面规划

## 1. 文档目的

本文用于指导“参数中心”聚合页面的产品设计、数据语义确认和后续实现。

页面需要在同一工作区内组织以下四类核心数据：

- 设备类型
- 设备型号
- 型号绑定的参数组
- 参数组包含的参数项

同时需要支持：

- 上述数据的高效浏览、检索和增删改查
- 型号与参数组之间的绑定、解绑
- 参数组被哪些型号复用的反向查看
- 某型号对应设备位于哪些设备组中的关联查看
- 参数项取值类型、取值空间和校验约束的清晰配置

本文不要求一次完成所有后端模型调整，但页面结构必须能承载最终模型，避免反复推翻。

## 2. 当前数据关系

当前主要关系如下：

```text
设备类型 1 ── N 设备型号
设备型号 N ── N 参数组（通过型号参数组绑定）
参数组   1 ── N 参数项
设备型号 1 ── N 设备
设备     N ── N 设备组（通过设备组设备绑定）
```

必须区分两种操作语义：

- `解绑参数组`：只删除当前型号与参数组的关系，不删除参数组及参数项。
- `删除参数组`：删除参数组实体，会影响所有引用该参数组的型号，属于高风险操作。

当前页面将这两种动作都表现为“删除”会造成误操作，后续界面必须明确区分。

## 3. 设计原则

### 3.1 型号是主工作上下文

用户通常先确定设备型号，再维护该型号使用的参数。因此页面应以“当前型号”为稳定上下文，而不是把参数组作为独立主列表。

### 3.2 关系操作与实体操作分离

绑定、解绑属于关系维护；新增、编辑、删除属于实体维护。按钮名称、确认提示和权限点都应分开。

### 3.3 高频操作留在主界面

型号切换、参数组切换、参数项查找和常用编辑不能依赖多层弹窗。复杂表单使用右侧抽屉，保持主列表上下文不丢失。

### 3.4 共享状态必须可见

参数组可能被多个型号引用。列表中应直接显示引用型号数量；编辑共享参数组前必须提示影响范围。

### 3.5 页面不复制后端业务规则

唯一性、引用保护、租户边界和取值约束应由后端最终校验。前端负责提前提示和提供适合的数据输入控件。

## 4. 推荐页面结构

推荐采用“左侧上下文工作台 + 右侧参数项工作区”的两栏布局。左侧上部放设备类型和型号，左侧下部放当前型号的参数组；右侧完整留给参数项浏览和维护。

相比三栏布局，该方案能为参数项表格保留约 70% 至 80% 的页面宽度。参数项需要同时展示名称、参数键、值类型、默认值、约束、单位和操作，是页面中横向信息密度最高的区域，应优先获得宽度。

```text
┌──────────────────────────────┬──────────────────────────────────────────────┐
│ 设备类型 / 设备型号           │ 当前参数组的参数项                           │
│ [搜索类型或型号]              │ 工业网关 A1 / 通讯参数                       │
│                              │ [搜索] [值类型] [必填]       [新增参数]      │
│ ▾ 工业网关                   │                                              │
│   ● 网关 A1                  │ 参数名  参数键  值类型  约束  默认值  单位   │
│     网关 A2                  │ MQTT    ...     文本    必填  ...     -      │
│ ▸ 打印设备                   │ 心跳    ...     整数    5~60  30      秒    │
│                              │                                              │
│ [+ 类型] [+ 型号]            │ [详情] [编辑] [复制] [删除]                  │
│──────────────────────────────│                                              │
│ 当前型号的参数组              │                                              │
│ 网关 A1       [绑定] [新建]  │                                              │
│ ● 通讯参数          8 项     │                                              │
│   设备配置          6 项     │                                              │
│   业务参数          4 项     │                                              │
└──────────────────────────────┴──────────────────────────────────────────────┘
```

建议尺寸和滚动方式：

- 左侧工作台宽 300 至 360px，允许拖动调整宽度，并可整体折叠。
- 左侧上下区域之间使用可拖动分隔条，默认类型/型号占 55%，参数组占 45%。
- 左侧两个区域独立滚动，标题和操作栏保持固定。
- 左侧折叠后保留展开按钮及“当前型号 / 当前参数组”简要提示。
- 右侧参数区占据剩余空间，只有参数表格内部允许横向滚动。
- 小屏幕下左侧工作台改为抽屉，右侧参数项保持主视图。

## 5. 各区域和弹框详细设计

### 5.1 左侧上部：设备类型和型号导航

采用分层树，设备类型为父节点，设备型号为子节点。

功能：

- 按类型名、型号名搜索，搜索时保留父子上下文。
- 单击型号切换主工作上下文。
- 类型节点显示型号数量。
- 型号节点显示已绑定参数组数量；异常配置可显示警告标记。
- 标题栏提供新增设备类型、新增设备型号和折叠按钮。
- 节点菜单提供编辑、删除等低频操作。

交互约束：

- 设备类型节点可展开，但不作为参数组查询条件。
- 新建型号时默认带入当前设备类型。
- 删除类型前必须检查是否仍有型号。
- 删除型号前必须显示设备数量、参数组绑定数量及影响说明。

### 5.2 左侧下部：型号参数组

下部标题栏固定显示当前型号名称，并提供绑定、新建和更多操作。未选择型号时，该区域显示明确空状态，不加载全部参数组。

参数组使用紧凑列表，每行至少显示名称、编码、类型、参数项数量、引用型号数量、是否允许编辑和版本。第一行显示名称和参数项数量，第二行显示编码、类型、版本及共享标记。当前参数组使用明确选中态。

主要操作：

- `绑定参数组`：从已有参数组库中选择一个或多个并绑定当前型号。
- `新建并绑定`：创建参数组后立即绑定当前型号。
- `关联浏览`：查看该参数组对应的型号、设备实例和设备组分布。
- `解绑`：只删除当前型号与参数组的关系。
- `编辑参数组`：编辑共享实体，打开前显示影响范围。
- `复制为新参数组`：复制参数组及其参数项，并仅绑定到当前型号。
- `删除参数组`：仅在无其他型号引用或用户完成高风险确认后提供。

常用的“关联浏览”和“解绑”可以直接显示为图标按钮，其余操作进入行尾菜单。不要在主列表中只放一个含义模糊的“删除”按钮。

### 5.3 参数组操作弹框

#### 5.3.1 绑定已有参数组

使用宽模态框，主体为可多选表格，支持按名称、编码和类型搜索。表格显示参数组名称、编码、类型、参数项数量、引用型号数量、版本和创建租户。已绑定当前型号的行置灰并标注“已绑定”。

确认后调用批量绑定接口；接口尚未提供时，可并发调用现有创建关系 API，但必须汇总成功项与失败项，不能在部分失败后显示笼统的成功提示。

#### 5.3.2 新建并绑定、编辑参数组

使用右侧抽屉，宽度建议 520 至 640px。字段包括参数组编码、名称、类型、是否允许编辑、版本和描述。

新建成功后立即绑定当前型号，并自动选中新参数组。编辑共享参数组前，抽屉顶部显示“该参数组被 N 个型号引用”，并提供查看引用明细的入口。

#### 5.3.3 参数组关联浏览

点击参数组行的“关联”按钮打开宽模态框，宽度为视口的 75% 至 85%，最大宽度约 1200px。弹框标题显示参数组名称、编码和版本，顶部显示引用型号数、设备实例数和设备组数。

弹框包含三个数据视图：

- `关联型号`：设备类型、型号名称、型号说明、租户；当前型号明确标记。
- `设备实例`：设备编码、设备名称、型号、序列号、使用状态。
- `设备组分布`：设备组名称、类型、租户、属于该组的相关设备数量；展开后查看设备明细。

三个视图可以使用页签，因为它们是同一弹框内互斥的关联明细，不会隐藏主页面筛选条件。所有列表使用服务端分页。弹框提供“仅当前型号”开关，便于缩小范围。

关联范围定义为：先找到引用该参数组的全部型号，再找到这些型号的设备，最后通过设备组设备关系聚合设备组。

#### 5.3.4 解绑参数组

使用小型确认框，文案明确为“从型号 网关 A1 解绑参数组 通讯参数”，并说明参数组和参数项不会被删除。确认后只删除型号参数组关系。

#### 5.3.5 删除参数组

删除属于高风险实体操作，使用二次确认模态框，而不是普通气泡确认：

- 显示引用型号数量和参数项数量。
- 有引用时默认禁止删除，优先引导用户解绑。
- 允许强制删除时必须输入参数组编码确认，并由后端再次校验权限和引用关系。
- 删除成功后清空右侧参数项并选择相邻参数组。

#### 5.3.6 复制参数组

使用中型模态框，允许填写新编码、新名称和版本，并选择是否复制全部参数项。默认勾选“复制后绑定当前型号”。复制必须由后端事务接口完成，避免出现只复制了部分参数项的状态。

### 5.4 右侧：参数项工作表

右侧是主要操作区域，使用可排序、可筛选的数据表格。

建议列：参数名称、参数键、值类型、默认值、取值约束摘要、单位、必填、来源或覆盖状态以及操作。

表格功能：

- 参数名称、参数键全文搜索。
- 按值类型、必填状态筛选。
- 支持详情、编辑、复制和删除。
- 参数较多时使用服务端分页，不在浏览器中固定拉取 500 条后自行分页。
- 后续可增加批量导入、导出和排序功能，但不作为第一阶段阻塞项。

### 5.5 参数项 CRUD 弹框

#### 5.5.1 新增参数项

使用右侧抽屉，宽度建议 560 至 680px。标题同时显示所属型号和参数组，防止用户在错误上下文中保存。

表单分为三个区域：

- 基本信息：参数名称、参数键、值类型、单位、是否必填、备注。
- 默认值：根据值类型显示对应输入控件。
- 取值约束：根据约束类型动态显示范围、枚举、正则或长度设置。

保存后提供两种结果：“保存”关闭抽屉、刷新当前页并定位新记录；“保存并继续新增”清理字段并保持当前型号和参数组上下文。

#### 5.5.2 参数项详情

使用只读抽屉，字段按基本信息、默认值与约束、关联范围分组。除原始字段外，还显示所属参数组、引用该参数组的型号数量和最近修改信息。权限允许时可通过“编辑”按钮原位切换为编辑模式。

#### 5.5.3 编辑参数项

复用新增抽屉和校验组件。参数键修改属于高风险操作，应提示可能影响设备侧协议映射。若参数组不允许编辑，抽屉保持只读并说明原因。

#### 5.5.4 复制参数项

复用新增抽屉并预填原记录，但清空 ID；参数键自动增加可编辑后缀。允许选择目标参数组，默认仍为当前参数组。保存前检查目标组内参数键唯一性。

#### 5.5.5 删除参数项

使用确认模态框，显示参数名称、参数键、所属参数组，以及该参数组引用的型号数量。删除只删除参数项，不删除参数组或绑定关系。若后续存在设备级参数值，应由后端返回引用冲突并禁止直接删除。

#### 5.5.6 表单校验和错误处理

- 参数键必填，并在当前参数组内唯一。
- 默认值必须符合值类型和取值约束。
- 范围约束必须校验上下界顺序。
- 长度约束必须是正整数，且字符串默认值不能超过最大长度。
- JSON 默认值必须通过前端格式校验，后端保存时再次校验约束配置 JSON。
- 保存失败时保留用户输入，将字段级错误定位到具体控件。
- 关闭有改动的抽屉前弹出放弃修改确认。

### 5.6 型号详情与关联浏览

左侧型号区域提供独立的 `型号详情` 按钮，打开宽抽屉。该抽屉展示型号自身的整体关联；参数组行上的“关联浏览”只展示该参数组的影响范围，两者不可混为一个入口。

型号详情抽屉包含型号基本信息、参数概览、设备实例、设备组分布和参数组引用。设备组分布的数据链路为：

```text
当前型号 -> 设备 -> 设备组设备关系 -> 设备组
```

现有通用列表 API 可以拼装该结果，但请求次数和前端聚合成本较高。数据量增大前应增加面向页面的聚合查询接口。

## 6. 编辑表单与取值约束

原 `parameter_type` 同时表达两类不同概念：

- 值类型：`BOOL`、`STRING`、`JSON`
- 约束或比较方式：`EQ`、`MIN`、`MAX`、`BETWEEN`、`REGEX`、`CONTAINS`

这会导致表单无法准确表达“数值，范围 5 到 60”或“字符串，限制最大长度”等常见规则。第四阶段已将模型直接替换为：

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `value_type` | 数据类型 | NUMBER、BOOL、STRING、JSON |
| `default_value` | 字符串形式默认值 | `30` |
| `constraint_type` | 约束类型 | RANGE、LENGTH、NONE |
| `constraint_config` | JSON 文本约束 | `{"min":5,"max":60}` |
| `unit` | 单位 | 秒、摄氏度、百分比 |
| `required` | 是否必填 | true |

前端根据 `value_type` 和 `constraint_type` 动态显示控件：

- BOOL：布尔默认值。
- NUMBER：最小值、最大值和单位。
- STRING：最大长度。
- JSON：按字符串存储，保存前校验 JSON 格式。

本轮不保留旧字段，也不提供旧数据迁移方案；初始化和测试数据直接使用新结构。

## 7. 关键业务规则

### 7.1 唯一性

- 同一租户下参数组编码唯一。
- 同一参数组内参数键唯一。
- 同一型号不能重复绑定同一参数组。
- 同一设备类型下型号名称是否唯一，需要产品确认并在后端约束。

### 7.2 参数组共享范围

参数组使用 `share_scope` 表达共享策略：

- `NOT_SHARED`：不共享，参数组只能绑定一个设备型号。
- `DEVICE_GROUP_SHARED`：设备组共享，只允许在指定设备组范围内复用。
- `GLOBAL_SHARED`：全局共享，允许在当前租户可访问范围内被多个型号复用。

当前数据模型只有“型号-参数组绑定”，尚无“参数组-设备组”直接关系。`DEVICE_GROUP_SHARED` 本轮先完成字段存储和界面表达；开始实现绑定校验前，必须增加参数组与设备组的明确关系，或定义可稳定计算的设备组归属规则。不能仅根据设备实例当前所在设备组临时推导，否则设备移动分组后会改变参数组共享合法性。

### 7.3 删除和级联规则

- 删除设备类型前检查型号引用。
- 删除型号前检查设备实例与参数组绑定。
- 删除参数组允许级联删除全部型号绑定关系和参数项；确认框必须显示级联影响数量。
- 参数组删除由数据库外键级联保证原子性，前端不得逐条删除子记录。
- 删除参数项前提示影响的参数组和型号。
- 仍应保留“解绑”作为不删除参数组的低风险操作。

### 7.4 租户边界

- 类型、型号、参数组、参数项和绑定关系必须处于可访问租户范围内。
- 绑定选择器只能列出与当前型号租户兼容的参数组。
- 平台租户与普通租户之间的可见、可复制、可绑定规则需要沿用现有后端租户策略，不能只在前端判断。

### 7.5 并发与版本

参数组已有 `version` 字段，但当前更新流程未体现乐观锁语义。若参数组允许被多型号共享，应明确：

- `version` 是业务版本还是并发版本。
- 编辑后是否自动递增。
- 是否需要复制为新版本，避免修改影响已投产型号。

## 8. 页面状态与 URL

建议将当前上下文写入查询参数：

```text
/xdev/parameter-center?modelId=8212&groupId=8921
```

这样可以：

- 刷新后恢复位置。
- 从设备型号、设备详情等页面直接跳转。
- 分享具体型号和参数组的管理链接。

页面需要完整处理以下状态：

- 初次进入但未选择型号。
- 设备类型或型号为空。
- 型号未绑定任何参数组。
- 参数组没有参数项。
- 绑定关系指向已删除或不可见资源。
- 当前资源只读或无操作权限。
- 查询失败、局部刷新失败和保存冲突。

## 9. API 与后端能力评估

第一阶段可以复用现有 CRUD API：

- 设备类型 CRUD
- 设备型号 CRUD
- 参数组 CRUD
- 参数项 CRUD
- 型号参数组绑定 CRUD

为提高效率和保证操作原子性，建议后续增加面向聚合页面的接口：

### 9.1 型号参数配置快照

一次返回型号摘要、绑定参数组、参数项计数和引用计数，避免页面并发拉取并在前端拼装。

### 9.2 批量绑定与解绑

支持一个事务内批量绑定或解绑参数组，并返回冲突项。

### 9.3 参数组复制

在后端事务内复制参数组、参数项并创建型号绑定。

### 9.4 型号关联概览

返回型号对应设备、设备组分布和参数组引用情况，供详情抽屉使用。

### 9.5 参数组关联概览

按参数组返回引用型号、相关设备、设备组聚合结果和数量摘要，并支持“全部引用型号 / 仅指定型号”两种范围。设备和设备组明细必须支持服务端分页，避免前端拉取全部关系后聚合。

### 9.6 引用保护

删除接口应返回结构化引用冲突信息，不能只依赖前端提前查询。

## 10. 权限设计

页面入口可以继续组合现有资源权限，但按钮必须按具体操作码控制：

- 设备类型：view/create/edit/delete
- 设备型号：view/create/edit/delete
- 参数组：view/create/edit/delete
- 参数项：view/create/edit/delete
- 型号参数组绑定：view/create/edit/delete
- 设备和设备组关系：详情抽屉只需要 view

“解绑”应使用型号参数组绑定的 delete 权限，而“删除参数组”应使用参数组 delete 权限。二者不能共用一个前端判断。

## 12. 第一轮实现验收标准

- 用户能在不离开页面的情况下从设备类型定位到设备型号。
- 选择型号后只显示真实绑定的参数组。
- 选择参数组后只显示该组的参数项。
- 切换型号或参数组时，选择状态和 URL 保持一致。
- 参数组列表能区分“解绑”和“删除参数组”。
- 左侧上下区域可以独立滚动，左侧整体可以折叠以扩大参数表格。
- 参数项列表使用服务端分页，不固定拉取全部数据。
- 用户可以从参数组行进入关联弹框，查看关联型号、设备实例和设备组分布入口。
- 新增、详情、编辑、复制和删除参数项都有明确且一致的弹框入口。
- 无权限、无数据、加载失败时均有明确且不误导的界面状态。
- 页面在常用桌面分辨率下无需横向滚动整个页面；仅参数项表格内部允许横向滚动。

## 13. 需要优先确认的产品决策

开始关系维护和 CRUD 实现前，需要确认以下事项：

已确认：

- 参数组通过 `share_scope` 区分不共享、设备组共享和全局共享。
- 删除参数组时级联解绑型号并级联删除参数项。
- 参数组 `version` 暂不自动递增或执行版本复制。
- `DEVICE_GROUP_SHARED` 本轮暂不实现约束，后续结合实际业务具体化为其他枚举值。
- 共享参数组被直接编辑前必须显示引用范围并要求用户确认影响。
- 参数项支持数值、布尔、字符串和 JSON 四类值；数值支持取值区间，字符串支持长度限制，JSON 按字符串存储并进行 JSON 格式校验。
- 平台租户创建的全局共享参数组允许普通租户绑定或复制，后端仍需分别校验可见性和目标租户边界。
- 设备类型和设备型号继续保留独立菜单，参数中心作为聚合维护入口，不取代独立页面。

上述决定作为阶段二及后续参数约束模型设计的当前基线；若共享范围枚举调整，需要同步 schema、proto、OpenAPI、生成配置、前端枚举和迁移策略。
